# 0.2.4 ‚Äî 2026-05-30

## Added

- **`openstreetmap_geocode` ‚Äî `effectiveQuery` enrichment**: the tool now calls `ctx.enrich({ effectiveQuery })` with the query string actually sent to Nominatim ‚Äî the free-form `query` value, or a comma-joined reconstruction of the structured address fields. Reaches both `structuredContent` and `content[]`. ([#12](https://github.com/cyanheads/openstreetmap-mcp-server/issues/12))
- **`openstreetmap_query_bbox` / `openstreetmap_query_nearby` ‚Äî `result_too_large` error contract**: both tools now declare `result_too_large` in their `errors[]` (Overpass can emit an out-of-memory remark on any query type). ([#10](https://github.com/cyanheads/openstreetmap-mcp-server/issues/10))

## Fixed

- **`openstreetmap_query_nearby` / `openstreetmap_query_bbox` ‚Äî `tag_key` without `tag_value` now rejected**: `resolveTagInput` treated a missing `tag_value` as valid and emitted a `["key"=""]` Overpass filter. Both tools now surface an `invalid_tag` error with an actionable hint instead. ([#9](https://github.com/cyanheads/openstreetmap-mcp-server/issues/9))
- **Overpass tools ‚Äî `data.recovery.hint` now populated for service-layer errors**: `query_timeout`, `result_too_large`, `rate_limited`, and `query_error` were thrown directly from the service without going through `ctx.fail()`, leaving `data.recovery.hint` empty. Each Overpass tool now catches the `McpError` from the service and re-throws via `ctx.fail(reason, ‚Ä¶)` so the error contract's recovery hint is set. ([#10](https://github.com/cyanheads/openstreetmap-mcp-server/issues/10))
- **Overpass ‚Äî HTTP 400 surfaced as `ValidationError` instead of `ServiceUnavailable`**: `fetchWithTimeout` now handles 400 before the service's manual check runs and throws `InvalidParams`; the tool layers remap this to `query_error` (`ValidationError`) so clients know the query itself is broken and should not retry. ([#11](https://github.com/cyanheads/openstreetmap-mcp-server/issues/11))
- **Overpass ‚Äî query-deterministic failures no longer retried**: `query_timeout`, out-of-memory (`result_too_large`), and HTTP 400 failures are now classified as non-transient via `isTransientOverpassError` passed to `withRetry`. Each now fails on the first attempt instead of burning 4 retry attempts against a public shared endpoint. A per-attempt `fetchWithTimeout` deadline (90 s) bounds each request. ([#13](https://github.com/cyanheads/openstreetmap-mcp-server/issues/13))
